# Contains Duplicate 學習筆記

> 學習日期：2026-01-06

---

## 這是什麼？（一句話白話解釋）

用 Set 的「不能重複」特性，邊走邊記檢查有沒有見過同樣的數字。

---

## 用什麼比喻理解？

### 比喻 1：簽到表

- **笨方法（O(n²)）**：每個人進來，都跟已經簽到的每個人比對名字
- **聰明方法（O(n)）**：用一本簽到簿，進來先查名字在不在，不在就簽上去

### 比喻 2：DevOps 場景

檢查 hostname 清單有沒有重複：
```
hostnames = ["web-01", "web-02", "db-01", "web-01", "cache-01"]
```
- 不用每台都跟每台比
- 用 set 記錄「見過的 hostname」，遇到重複立刻發現

### 比喻 3：Set vs Dict

| 資料結構 | 用途 | 比喻 |
|---------|------|------|
| Dict | key-value 配對 | 電話簿（名字 → 號碼）|
| Set | 只記錄 key 存不存在 | 簽到表（只記誰來過）|

---

## 踩過什麼雷？

### 雷 1：Set 加入元素的語法

```python
# ❌ 錯誤
my_set += i     # TypeError: unsupported operand type(s) for +=: 'set' and 'int'

# ✅ 正確
my_set.add(i)   # 用 .add() 方法加入單一元素
```

> 記法：List 用 `.append()`，Set 用 `.add()`

---

### 雷 2：把「建立 set」跟「查詢 set」搞混

```python
# 查詢單一元素 → O(1)
if x in my_set:
    ...

# 建立整個 set → O(n)，要遍歷整個陣列！
my_set = set(nums)
```

> 重點：`set(nums)` 看起來一行，但背後還是要跑 O(n)

---

## 兩種解法

### 解法一：邊走邊記（推薦面試用）

```python
def contains_duplicate(nums: List[int]) -> bool:
    my_set = set()
    for i in nums:
        if i in my_set:        # 先查：見過嗎？
            return True        # 見過 → 有重複！
        my_set.add(i)          # 沒見過 → 記下來
    return False               # 全部看完都沒重複
```

### 解法二：一行 Pythonic 寫法

```python
def contains_duplicate(nums: List[int]) -> bool:
    return len(nums) > len(set(nums))
```

> 原理：set 會自動去重，如果長度變短了就代表有重複

---

## 複雜度分析

| 解法 | 時間複雜度 | 空間複雜度 | 最佳情況 |
|------|-----------|-----------|---------|
| 暴力解（雙層迴圈） | O(n²) | O(1) | O(n²) |
| 邊走邊記 | O(n) | O(n) | O(1) — 第 2 個就重複 |
| 一行 Pythonic | O(n) | O(n) | O(n) — 還是要跑完 |

---

## Trade-off：什麼時候用哪種？

| 情境 | 建議用 | 原因 |
|------|--------|------|
| 資料量大、可能提早發現重複 | 邊走邊記 | 可以 early return |
| 資料量小、追求可讀性 | 一行寫法 | 代碼簡潔，團隊好維護 |
| 面試 | 先講邊走邊記 | 展示你懂 hash table 原理 |

---

## Edge Case

| 輸入 | 預期輸出 | 原因 |
|------|---------|------|
| `[]` | `False` | 空陣列沒有元素，不可能重複 |
| `[1]` | `False` | 只有一個元素，不可能重複 |
| `[1, 1]` | `True` | 兩個一樣，重複了 |

---

## 面試怎麼回答？

> 「Contains Duplicate 可以用 Set 解。遍歷陣列時，對每個數字先檢查它有沒有在 Set 裡面。如果在，代表之前見過，就回傳 True；如果不在，就加進 Set。這樣時間複雜度是 O(n)，空間複雜度也是 O(n)。」
>
> 「另外也可以用一行 Pythonic 寫法：比較原陣列長度和轉成 Set 後的長度。但這種方法沒辦法 early return，大資料量時效能較差。」

## 模擬面試問答紀錄

### 問題 1：為什麼用 Set 不用 Dict？

> **我的回答**：因為只需要知道「有沒有見過」，不需要存額外資訊。Set 只存 key，語意更清楚、更省空間。

---

### 問題 2：`set(nums)` 是 O(1) 還是 O(n)？

> **我的回答**：O(n)！雖然只有一行，但 Python 背後要遍歷整個陣列才能建立 Set。只有「查詢單一元素」`x in my_set` 才是 O(1)。

---

### 問題 3：兩種方法複雜度一樣，怎麼選？

> **我的回答**：
> - 資料量大、可能提早找到重複 → 用邊走邊記（可以 early return）
> - 資料量小、追求可讀性 → 用一行寫法（團隊好維護）
> - 面試時先講邊走邊記，再提一行寫法，展示你兩種都會

---

### 問題 4：空陣列會回傳什麼？

> **我的回答**：回傳 False。邊走邊記的話 loop 跑 0 次，直接 return False。一行寫法的話 `len([]) > len(set([]))` = `0 > 0` = False。

---

## 學到的核心觀念

1. **Set 是簡化版的 Dict**：只需要「存不存在」時，用 Set 更簡潔
2. **先查再存**：跟 Two Sum 一樣的模式
3. **建立 vs 查詢**：`set(nums)` 是 O(n)，`x in set` 是 O(1)
4. **Trade-off 思維**：效能 vs 可讀性，要看情境選擇
