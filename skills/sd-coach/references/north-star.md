# North Star

## Win Condition

通過大廠 system design 面試:在 turn-based mock 中無提示跑完 4-step framework,講得出每個 trade-off 的 why,達 Tier 3 scorecard 門檻。

**Derivation:**
- SD 面試測的是 mental model 廣度 + trade-off 推理,不是 CS 理論深度。「以理解為本」同時是手段和最高 ROI:背 65 題標準答案不可遷移;打穿 building blocks + 分散式原理,沒練過的題也能現場組合出來。
- 每堂課都要產出 interview-ready artifacts(one-liner、how-to-say-it、練過的回答),不能只停在「懂了」。學了沒轉成面試語言,等於面試房間裡不存在。
- Production/DevOps 視角(monitoring、failure modes、營運成本)是這個學員的差異化武器,每個設計都帶到。
- 家族分工:k8s/SRE 深度外包給 `k8s-coach`,coding/DSA 外包給 `leetcode-coach`,本 coach 不重造。

## Tie-Break Rule

當「更深的理論」與「更廣的 mental model / retention / mock 練習」競爭 session 時間,**面試 ROI 贏**:深度以 interview depth 封頂(機制 + trade-off + when-to-use),省下的時間投給複習與 mock,那裡才有複利。

**Derivation:**
- 正式證明與數學推導花時間,在面試房間裡換不到分。engine 的 Depth Ceiling(Three Questions)由教練主動執行,不等學員自己痛。
- Park 掉的線索進 curiosity branch,永遠拉得回來;session 時間拉不回來。
- 覆蓋率同理:一題打穿勝過三題略過。深挖的單位是 building block,不是枝節理論。
