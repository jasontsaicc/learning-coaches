# North Star

## Win Condition

通過 AWS ProServe Delivery Consultant (Cloud Architect, Taiwan) 面試 loop:給一個沒看過的 migration 或 hybrid architecture 案例,學員能用英文在 hire bar 交出結構化的答案。

**Derivation:**
- 交付物要含四塊:Assess / Mobilize / Migrate & Modernize 的分階段計畫、逐 workload 的 7R 判斷(每個決定講得出 why)、hybrid networking 設計、以及 Well-Architected 五大支柱的 trade-off。這四塊是 ProServe 面試官實際打分的維度,少一塊就露餡。
- 「講得出 why」是重點:7R 不是背菜單,是能對著這個 workload 的限制(licensing、耦合度、時程、成本)推理出選 rehost 還是 replatform。這是可遷移的判斷力,換一個沒練過的案例也組得出來。
- 要守得住 resume thread-pull:面試官會從履歷往下挖 EC2 / Linux 深度,學員得能接住,素材走 linux-interview-bank。這條是這個 loop 的隱藏 gate,不能只練架構層。
- 英文交付是硬條件,不是加分:這是 client-facing 顧問職,講不清楚等於做不到。

## Tie-Break Rule

delivery 贏 coverage。當「case / mock 練習」和「再多學一個 AWS service」搶 session 時間,**mock 贏**。

**Derivation:**
- 這個 loop 打分的是判斷力與溝通,不是 service 知識的百科廣度。多背一個服務的邊角參數,在面試房間裡換不到分;把一個案例講穿、被追問也守得住,才是複利。
- 第二層仲裁:一旦 phone screen 或 loop 日期確定,整週計畫繞著它重排,mock 往前提。日期是外部硬約束,學習進度可以讓路,面試窗口不會等人。
